Conectar a MySQL desde Visual Basic .NET

octubre 24, 2013 - 11:08
1 Comentario
Twittear

Uno de las cosas más comunes con las que nos encontramos al momento de desarrollar una aplicación es la de “conectarla” con el servidor de base de datos, obviamente para poder obtener o guardar datos en el.

Visual Studio provee asistentes para configurar una conexión a nuestro servidor y utilizarlos con los componentes GUI, pero hay ocasiones en las que nos veremos forzados a realizar esta conexión de manera manual, y es aquí donde entra la utilidad de este tutorial.

A continuación explicare como conectar el servidor de MySQL con Visual Basic.

Requisitos

Para poder conectar a MySQL es necesario contar con el conector, el cual nos brindará la API necesaria para poder realizar nuestras funciones de alta, baja, actualización y consultas, entre otras.

Esta librería la descargamos del sitio web de MySQL; una vez descargada la instalamos.

Configuración

Una vez instalada  es hora de importarla en nuestro proyecto de Visual Studio. Con el proyecto abierto, nos dirigimos al menú PROYECTO y elegimos el submenú Propiedades… En esta ventana hacemos clic en la pestaña Referencias y después en el botón Agregar… ahí elegimos la opción Referencia.

agregar referencia

En la ventana a lado izquierdo elegimos Ensamblados  y luego Extensiones. Dentro de las extensiones que aparecen buscamos la extensión Mysql.Data, la seleccionamos y activamos el check que le aparece  (verifica que la versión de la extensión sea la que descargaste e instalaste). Pulsamos Aceptar y cerramos la ventana.

agregar extension

Si la extensión no aparece listada, presiona sobre el botón Examinar… y procede a buscarla en el directorio donde instalaste el conector MySQL.

Una vez agregada la extensión, es momento de pasar al código. Importamos la librería a nuestro archivo para no tener que utilizar todo el namespace. Aquí dejare a tu elección crear tu propia clase o formulario.

Imports Mysql.Data.MySqlClient

Con esta línea estamos listos para realizar consultas al servidor MySQL.

Objetos del conector MySQL

Algunos de los objetos que se encuentran en la librería y con los que trabajaremos en este tutorial son los siguientes:

  • MySqlConnection: Representa la conexión al servidor  MySQL.
  • MySqlCommand: Representa la sentencia SQL que será ejecutada.
  • MySqlDataReader: En este objeto se almacenan los registros retornados por una consulta SQL.
  • MySqlException: La excepción que es lanzada cuando el servidor MySQL devuelve un error.

Para hacer entendible este tutorial haremos lo siguiente:

  • Una función llamada conectar(), la cual hará la conexión al servidor de base de datos.
  • Una función llamada consultar(), la utilizaremos para realizar consultas SELECT
  • Una función llamada ejecutar(), la utilizaremos para realizar sentencias INSERT, DELETE Y UPDATE.

Codigo

Creando la conexión al servidor MySQL.

Establecemos la cadena de conexión con la variable ConnectionString utilizando cuatro variables locales. El método Open() abre la conexión al servidor MySQL, lo declaramos dentro de un bloque TRY-CATCH en caso de que surja alguna excepción.

Private Sub conectar()

    Private servidor As String = "localhost"
Private baseDatos As String = "base_datos"Private usuario As String = "usuario_bd"
    Private password As String = "password_bd"

    Try

    conexion.ConnectionString = "Server=" & servidor _
        & "; Database=" & baseDatos _
        & "; Uid=" & usuario _
        & "; Pwd=" & password

    conexion.Open()

    Catch ex As Exception
        MsgBox(ex.Message)
    End Try

End Sub

Desconectando el servidor MySQL

Siempre es aconsejable que después de realizar cualquier consulta a la base de datos desconectar la conexión, es por eso que declaramos esta función que se encargara de desconectar.

Public Sub desconectar()

    Try
        'Cerrar la conexion si esta abierta
        If conexion.State = ConnectionState.Open Then

            conexion.Close() 'Cerrar la conexion
            conexion.Dispose() 'Liberar recursos

        End If

    Catch ex As Exception
        MsgBox(ex.Message)
    End Try

End Sub

Realizando consultas SELECT

Esta función se encargará de realizar las consultas SELECT. La función ExecuteReader() del objeto MySqlCommand ejecuta una consulta que espera devolver filas,  y para ello se utiliza una variable  de tipo MySqlDataReader para guardar los registros obtenidos.

Public Function consultar(ByVal sql As String) As MySqlDataReader

    Dim reg As MySqlDataReader = Nothing 'Guardara las filas de la consulta

    conectar() 'Conectar al servidor

    Try

        Dim query As New MySqlCommand(sql, conexion) 'Especificamos la sentencia MySQL
        reg = query.ExecuteReader() 'Ejecuta la sentencia MySQL

    Catch ex As Exception
        MsgBox(ex.Message) 'Muestra un mensaje de Error si lo hay
    End Try

    Return reg

End Function

Esta función devuelve un objeto MySqlDataReader.

Realizando sentencias INSERT, UPDATE Y DELETE

A diferencia de las consultas SELECT, las sentencias INSERT, UPDATE Y DELETE no devuelven filas (registros), en su lugar, devuelven un numero de filas afectadas.

La función ExecuteNonQuery() devuelve un numero entero indicando el numero de filas afectas por la sentencia.

Public Function alta(ByVal sql As String)

    conectar() 'Conectar a la BD
    Dim filas As Integer = 0 ‘Guardara las filas afectadas

    Try

    Dim query As New MySqlCommand(sql, conexion) 'Especifica la Sentencia MySQL
    filas = query.ExecuteNonQuery() 'Ejecuta la Sentencia MySQL

    Catch ex As Exception
        MsgBox(ex.Message) 'Muestra un mensaje de Error si lo hay
    End Try

    desconectar() 'Cerrar la Conexion a la BD

Return filas

End Function

Ejemplos

Leyendo los registros de una consulta SELECT

Para leer los datos de un objeto MySqlDataReader lo haremos de la siguiente manera.


conectar()'Conectar a la BD

Try

    Dim registros As MySqlDataReader = consultar("SELECT nombre FROM usuario WHERE usuario = 'juan' AND         password = sha1('123') ")

    If registros.HasRows = True Then

    'Llamar al metodo Read() para leer el resultado de la consulta
        While registros.Read()
            MsgBox("Hola: " & registros.GetString("nombre"))
        End While

    Else
        MsgBox("Usuario o Contraseña Incorrecta")
    End If

    registros.Close() 'Cerrar el DataReader

Catch ex As Exception
    MsgBox(ex.Message)
End Try

desconectar() 'Cerrar la conexion a la BD

La variable booleana HasRows nos indicara si la consulta tuvo resultados, si hubieron se accede a la fila leyéndola mediante un ciclo WHILE y como condición se llama al método Read() este devolverá falso cuando se hayan recorrido todas las filas de la consulta.

Para acceder a los campos de la fila (columnas) se llama al método que corresponda. Por ejemplo, si deseamos obtener una cadena de tipo STRING llamamos a la función GetString(“nombre”)  y especificamos el nombre del campo, en este caso, nombre.

Es recomendable cerrar el objeto MySqlDataReader después de cada consulta.

Insertando registros en la base de datos

Insertar, actualizar o eliminar datos es similar a realizar una consulta, solo que en este caso comparamos cuantas filas afectadas hubieron.

Dim sql As String = "INSERT INTO usuario VALUES('juan', sha1('123'), 'ADMIN' )"

If conexion.alta(sql) > 0 Then
    MsgBox("Datos Insertados Correctamente")
Else
    MsgBox("No se Insertaron Datos")
End If

Eliminar

Dim sql As String = "DELETE FROM usuario WHERE usuario = 'juan'"

If conexion.alta(sql) > 0 Then
    MsgBox("Datos Eliminado Correctamente")
Else
    MsgBox("No se Eliminaron Datos")
End If

Actualizar

Dim sql As String = "UPDATE usuario SET usuario = 'paco' WHERE usuario = 'juan'"

If conexion.alta(sql) > 0 Then
    MsgBox("Datos Actualizados Correctamente")
Else
    MsgBox("No se Actualizaron Datos")
End If

Conclusión

EL objetivo de este tutorial fue el de dar a conocer como conectar a MySQL desde Visual Studio y realizar algunas operaciones básicas.

Obviamente para hacer más dinámica la aplicación tal vez las sentencias SQL las tengas que concatenar con otras variables que recibas de formularios.
